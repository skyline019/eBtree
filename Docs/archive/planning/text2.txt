🔥 第一层：编译器与链接器（MSVC 专属“死锁解锁”）

Windows 下 MSVC 的默认配置极其保守，必须手动撬开：

优化项 操作（项目属性 -> C/C++） 预期收益
开启全程序优化（LTCG） /GL（C++） + /LTCG（链接器） 跨 .cpp 内联，函数调用开销降低 15%
强制 AVX2 向量化 /arch:AVX2 + /Qvec（必须手动加，否则 MSVC 不自动向量化） 你的 SIMD 摘要比对提速 2~3 倍
关闭安全检查 /GS- 移除栈 Cookie 检查，每条指令节省 2~3 个周期
启用内建函数 /Oi 让 memcpy/memset 直接变成 CPU 单指令，不调用 CRT
浮点激进模式 /fp:fast 允许重排浮点运算，提速 10%（如果你用浮点摘要）
静态链接运行时 /MT（你已确认） 消除 VCRUNTIME140.dll 加载延迟，且避免多版本冲突

链接器附加：/OPT:REF /OPT:ICF（去除冗余代码，缩小二进制并提速）。

🔥 第二层：内存分配器（砍掉 MSVC 堆锁）

MSVC 的 new/delete 在多线程下会全局串行化。你的 vShard 分片架构最适合用 HeapCreate(HEAP_NO_SERIALIZE) 创建私有堆。

```cpp
// 每个 vShard 拥有独立堆（无需锁争用）
HANDLE g_privateHeap = HeapCreate(HEAP_NO_SERIALIZE, 0, 0);

inline void* operator new(size_t size) {
    return HeapAlloc(g_privateHeap, 0, size);
}
inline void operator delete(void* ptr) {
    HeapFree(g_privateHeap, 0, ptr);
}
```

注意：HEAP_NO_SERIALIZE 要求单线程访问此堆。由于你有 256 个分片，可以给每个分片独立创建 Heap，写入时只锁分片，堆操作完全不进入内核态。

必做补充：使用 _aligned_malloc（在 <malloc.h> 中）分配内存映射缓冲区和 4KB 对齐的 I/O 缓冲区，避免 CRT 堆污染。

🔥 第三层：存储 I/O（解决 3.x TPS 的核心）

这是 Windows 与 Linux 差距最大的地方，绝对禁止使用 FlushFileBuffers。

优化项 实现方式 作用
直写绕过缓存 CreateFile 带上 FILE_FLAG_NO_BUFFERING \| FILE_FLAG_WRITE_THROUGH 让 WriteFile 返回即落盘，且不污染系统缓存
4KB 扇区对齐 所有 WriteFile 的 lpBuffer、偏移量、长度必须是 4096 的整数倍 否则 NO_BUFFERING 会报错 ERROR_INVALID_PARAMETER
预分配文件空间 SetFileValidData（需开启 SeManageVolumePrivilege）或 SetFilePointer + SetEndOfFile 避免写入时频繁更新文件大小元数据，降低延迟抖动
禁用索引服务 在文件创建时加 FILE_FLAG_OPEN_NO_RECALL（远程存储）及 FILE_ATTRIBUTE_NOT_CONTENT_INDEXED 防止 Windows Search 拦截文件写入，减少 CPU 中断

你的 BAD 平衡态落地：维护一个 4KB 对齐的内存缓冲池，攒够 4KB 或 512 条记录，一次性调用 WriteFile（此时返回即落盘），实现 5 万 TPS 持久化。

🔥 第四层：线程与调度（榨取多核）

优化项 API 效果
绑定 CPU 核心 SetThreadAffinityMask(GetCurrentThread(), 1 << core_id) 避免线程在核心间漂移导致 Cache Miss
提升 IO 线程优先级 SetThreadPriority(GetCurrentThread(), THREAD_PRIORITY_HIGHEST) 让刷盘线程优先获得 CPU 时间片，降低长尾延迟
使用原生线程池 CreateThreadpoolWork（或直接 std::thread + 绑定） Windows 线程池能减少创建/销毁开销

关键警告：Windows 调度器对 1000+ 线程敏感。务必保持工作线程数 ≤ CPU 核数（如 8 核就开 8 个线程）。349k TPS 的高并发应通过异步任务队列 + 工作线程池实现，而非创建 1000 个 std::thread。

🔥 第五层：内存映射（Windows 的 mmap 替代）

Windows 的 mmap 对应 CreateFileMapping + MapViewOfFile。针对 349k 压测，有 3 个专属优化：

优化项 代码 效果
启用大页（Large Pages） VirtualAlloc + MEM_LARGE_PAGES（需提前开启 SeLockMemoryPrivilege） 减少 TLB Miss，B-Tree 遍历速度提升 10~15%
指定视图基址 MapViewOfFileEx 手动指定起始地址 避免 ASLR 导致地址随机，有利于 CPU 分支预测
预读提示 PrefetchVirtualMemory（Win8+） 告诉内核提前加载 B-Tree 内部节点到物理内存

🔥 第六层：高精度计时（避开 std::chrono 陷阱）

std::chrono::steady_clock 在 MSVC 下调用 QueryPerformanceCounter 有系统调用开销。用 __rdtsc 替换：

```cpp
#include <intrin.h>
inline uint64_t now_cycles() { return __rdtsc(); }

// 假设 CPU 3.0GHz，2ms = 6,000,000 个周期
const uint64_t BAD_WINDOW_CYCLES = 6000000;
if (now_cycles() - last_flush_cycles > BAD_WINDOW_CYCLES) flush();
```
